# /pr-ready

Prepare the code for a Pull Request.

## Purpose
Check if code is ready for PR

## Usage
`/pr-ready`

## Speed
Moderate (2-4min)

## When to use
Before creating PR

## Instructions
1. Perform a final quality audit for logs and unused code.
2. Summarize the changes made.
3. Generate a Markdown PR description including a "Changes" list and "Commands Run" section.
4. Confirm /test-page-quick was successful.
