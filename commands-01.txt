# /cleanup-unused-code
Remove unused imports, commented-out code, unused variables, and console logs. Standardize the file structure after cleaning.
File: .cursor/commands/imports.md

Markdown

# /fix-import-paths
Fix broken import paths and standardize the order: React first, then external libraries, then internal components, and types last. Convert relative paths to absolute `@/` paths.

# /test-page-quick
Perform a rapid smoke test of the current page. Identify rendering errors, broken API calls, or console errors. Verify critical paths like Dashboard, Products, and Customers.

3. Workflow Chains (The "Power" Commands)
These files tell Cursor to run multiple steps in a specific order.

File: .cursor/commands/standardize.md

Markdown

# /standardize-page
Perform a full page standardization. 
1. Run /fix-import-paths.
2. Run /cleanup-unused-code.
3. Fix layout spacing and convert px to design tokens.
4. Standardize UI components (Filters, Tables, Forms).
5. Run /test-page-quick to verify.
Wait for approval after each step.
File: .cursor/commands/pr.md

Markdown

# /pr-ready
Prepare the code for a Pull Request.
1. Perform a final quality audit for logs and unused code.
2. Summarize the changes made.
3. Generate a Markdown PR description including a "Changes" list and "Commands Run" section.
4. Confirm /test-page-quick was successful.
